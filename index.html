1 - Development Environment with Docker and Portainer

Objective:

Create, in a home lab environment, a server to study containers and other applications available in this format.

Hardware and Software Description:

In this environment, I use an old computer with the following settings:

Hardware:

  • 4 core CPU;
  • 7 GB of RAM;
  • 120 GB of SSD;
  • 1 TB of HDD;
  • 1 Gbps network interface.

Software:

  • Linux Ubuntu Server;
  • Docker;
  • Portainer CE;
  • SSH Server.

Deployment:

I first install Ubuntu Server Linux. During the installation, I select to install the SSH Server and Docker.

Afterward, I set a static IP address for the server.

Then, I install Portainer Community Edition (CE) following the official instructions here.

To install Portainer, I create a volume to store the Portainer database with the following command:

docker volume create portainer_data

Next, I download and install the Portainer Server container with the following command:

docker run -d -p 8000:8000 -p 9443:9443 --name portainer --restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:latest

I decided to use a self-signed SSL certificate to secure port 9443.

Logging In:

I log into my local Portainer Server instance by opening a web browser and going to:

https://192.168.1.6:9443
